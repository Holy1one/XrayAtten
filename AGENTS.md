# AGENTS.md

## 基本要求

- 先阅读 README、项目结构和相关代码，不要盲目修改。
- 每次只做任务要求范围内的最小必要修改。
- 不要大规模重构，除非任务明确要求。
- 不要修改与任务无关的文件。
- 不要删除已有功能。
- 不要提交密钥、账号、token、密码、数据集等敏感文件。

## 执行要求

- 修改前先说明计划。
- 修改后运行相关测试或构建命令。
- 如果测试失败，先尝试修复。
- 最终总结必须包含：
  - 修改了哪些文件
  - 为什么这样修改
  - 运行了哪些验证命令
  - 是否还有潜在风险

## Git 要求

- 不要自动 push。
- 不要自动删除文件。
- 不要修改 .gitignore，除非任务明确要求。
